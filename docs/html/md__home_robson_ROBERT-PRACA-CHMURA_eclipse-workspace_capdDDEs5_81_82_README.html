DESCRIPTION OF THIS PROJECT

The repository contains source codes for the rigorous integration library for Delay Differential Equations (DDEs) of the form:

x'(t) = f(x(t), x(t-d1), ..., x(t-dm))

Where d1,..., dm are m delays. It is build upon ideas from CAPD library for ODEs (capd.ii.uj.edu.pl). By rigorous methods we understand routines that compute the enclosure (interval bounds) on the true solutions to the problems, which enable applying them in Computer Assisted Proofs. Please reffer to CAPD documentation for general papers on the topic. And see References for the ones specyfic to DDEs.

This projec requires the CAPD library, but the repository contains compatible CAPD version in './external/' directory. You can use your own version of the CAPD, but in that case you need to modify CAPDBINDIR and CAPDLIBDIR variables in the template Makefile scripts build-conf.mk and build-conf-mp.mk.

The work is open-source, under GPL license. Please consider citing some of the papers listed in References, when using this work in any scientific project.

The integrator is written as a template-based C++ library and heavily uses CAPD library routines for rigorous numerics. The main part, that is designed to be used separately, is contained in 'include/capd/ddes'. As of 2020 we aim to incorporate this part of the library in the core of CAPD DynSys project. For details about CAPD see http://capd.ii.uj.edu.pl . Files in 'include/capd/ddesHelper' are designed to be outside of the base implementation and they aim at providing simple algorithms to create, manipulate and refine initial data for the integrator.

This project also contains several programs under examples directory those programs can be build issuing the following command: 

make [program_name]

where [program_name] is equal to the name of any directory under 'examples/'. The resulting executable file will go to the bin/examples/[program_name] directory.

The most important part is the rigorous integrator for DDEs (DDE library). It is contained in include/capd/ddes. As of 2022, I have decided to reorganize it again to be more complaint with the CAPD structure. I have followed example of 'external/capd/capdDynSys4/include/capd/pdes' in this manner.

As of 2022 the integrator uses CAPD DynSys version 5.1.2.

COMPILATION OF THE LIBRARY OND CUSTOM PROGRAMS

If you have downloaded version without CAPD and want to use your own version of this library, then you need to modify file 'build-conf.mk' first. Follow the instructions in that file to make the Makefile work.

If you have version with bundled CAPD library (there is './external/capd' folder), then you need to build it first.

A script 'capd-build.sh' in './external' can do everything automatically: 

# first, build CAPD (it is a long process)
cd external
# add permission to helper scripts if neccessary
chmod a+x capd/configure
chmod a+x build.sh
./build.sh

Finally, you can check CAPD build by entering: 

cd external/capd_build
make check

In case of any errors in compiling CAPD library, please check requirements on http://capd.ii.uj.edu.pl, or contact me at rober.nosp@m.t.sz.nosp@m.czeli.nosp@m.na@u.nosp@m.j.edu.nosp@m..pl.

CAPD v. 5 AND SHARED LIBRARIES (UPDATED 2022)

In a CAPD v5 there is a problem with compiled program, when the library was installed localy. The system does not see the dynamically linked libraries.

From 'make install' of CAPD you get info:

Libraries have been installed in: /home/robson/workspace/capdDDEs5.1.2/bin/capd/lib

If you ever happen to want to link against installed libraries in a given directory, LIBDIR, you must either use libtool, and specify the full pathname of the library, or use the '-LLIBDIR' flag during linking and do at least one of the following:

  • add LIBDIR to the 'LD_LIBRARY_PATH' environment variable during execution
  • add LIBDIR to the 'LD_RUN_PATH' environment variable during linking
  • use the '-Wl,-rpath -Wl,LIBDIR' linker flag
  • have your system administrator add LIBDIR to '/etc/ld.so.conf'

See any operating system documentation about shared libraries for more information, such as the ld(1) and ld.so(8) manual pages.

For now, I do the following:

  • create file '/etc/ld.so.conf.d/CAPDv5.conf' with the LIBDIR in it:

    sudo cat '/home/robson/workspace/capdDDEs5.1.2/bin/capd/lib' > /etc/ld.so.conf.d/CAPDv5.conf

  • reload ldconfig:

    sudo ldconfig

This need admin rights. I think the option

  • use the '-Wl,-rpath -Wl,LIBDIR' linker flag

could work better, it can be implemented with Makefile. TODO: Try to implement this method instead of ldconfig.
