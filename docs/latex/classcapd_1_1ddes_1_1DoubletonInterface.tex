\subsubsection*{template$<$typename Matrix\+Spec, typename Policies\+Spec = capd\+::dynset\+::\+Id\+Q\+R\+Policy$>$\newline
class capd\+::ddes\+::\+Doubleton\+Interface$<$ Matrix\+Spec, Policies\+Spec $>$}

interface I use in my code to store doubleton data of the form\+:

X = x + C $\ast$ r0 + B $\ast$ r

I use storage$\ast$ to name methods, as for example \mbox{\hyperlink{classcapd_1_1ddes_1_1DoubletonInterface_a9b9e5b7337fb3f9370716a7ca0e299d7}{dimension()}} is usually related to the algebraic dimension of the space in C\+A\+PD. In my case I will have doubletons that represent data by a huge number of coefficients, but in small dimensional space, i.\+e. piecewise Taylor curves of order n in d -\/dimensional space. so the storage\+Dimension will be (n+1) $\ast$ d;
