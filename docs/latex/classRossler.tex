\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Param\+Spec$>$\newline
class Rossler$<$ Param\+Spec $>$}

This class defines f in the right hand side of the D\+DE (1) x\textquotesingle{}(t) = f(x(t), x(t-\/tau), ...)

Here, we define R"ossler O\+DE that does not depend explicitly on the past\+:

(2) x\textquotesingle{}(t) = f(x(t))

If you are doing your own stuff, the R\+HS of your equations should follow this exemplary definition.

It is a little harder than in the C\+A\+PD itself, as C\+A\+PD allows for specifying R\+HS as a simple string. Here, the approach is similar to specifying R\+HS as a function Node f(\+Node t, Node in\mbox{[}$\,$\mbox{]}, int in\+Dim, Node out\mbox{[}$\,$\mbox{]}, int out\+Dim, Node params\mbox{[}$\,$\mbox{]}, int par\+Dim) used for example in ./external/capd/capd\+Dyn\+Sys4/examples/poincare/\+I\+Poincare\+Map\+Example.cpp

This does not need to be template class, but I like to do it that way so that i can use it for rigorous and non-\/rigorous computations!

computation of R\+HS of the equation x\textquotesingle{}(t) = f(x(t), x(t-\/tau), ...),

It must be template, as we use autodiff to generate derivatives of this.

Let d = \mbox{\hyperlink{classRossler_a635c91889bf383183c05c02219c95840}{image\+Dimension()}}. We follow the convention that In\+Vector\+Spec x = (x\+\_\+1(t), x\+\_\+2(t), ..., x\+\_\+d(t), x\+\_\+1(t-\/tau), x\+\_\+2(t-\/tau), ..., x\+\_\+d(t-\/tau), ...) Therefore in code below, e.\+g. x\mbox{[}1\mbox{]} is y coordinate in the original \mbox{\hyperlink{classRossler}{Rossler}} O\+DE. (Please remember, that indices start from 0 in In\+Vector\+Spec x)

The t is current time for non-\/autonomous equations. The fx parameter (output), fx means \textquotesingle{}f(x)\textquotesingle{}, is the output of the operator. It is for convenience of calling operator (as the return type cannot be auto-\/deduced in C++!), and also for avoiding excessive copy-\/consttructing, move-\/copying, etc. It just alter the value of already existing vector given by reference.
